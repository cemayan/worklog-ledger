How it works

Worklog Ledger uses your existing Jira Cloud browser session (or an API token you provide) to read and write worklogs directly — the same way Jira's own UI does. Settings, client mappings and rates are stored in your browser's extension storage. Nothing touches a third-party server.

Built to be trusted

Extensions that touch your Jira deserve scrutiny, so Worklog Ledger keeps the surface auditable:

  • Open source — the entire extension is public at github.com/cemayan/worklog-ledger. What you install is what you can read.
  • No backend — your worklogs, tokens and rates never leave your browser. The only permission it asks for is *.atlassian.net.
  • One honest exception — if you buy the report plan, the extension makes a single license check that contains just your purchase email. Never any Jira data.
  • No analytics, no tracking — see the privacy policy.

FAQ

How does the license work?

After checkout, open the extension's Settings and enter the email you used at purchase — paid features unlock instantly. No account, no password.

How do I cancel or manage my subscription?

Use the link in your Paddle receipt email (Paddle is our payment provider), or just write to support@cemayan.com and we'll handle it. Cancelling keeps paid features until the end of the billing period.

Is my Jira data safe?

It never leaves your browser. The extension talks only to your own Jira Cloud site; there's no backend and no analytics. The code is open source.

Tempo? Jira Data Center?

Not yet — the extension reads native Jira Cloud worklogs today. Tempo interop and Data Center support are on the roadmap; tell us if you need them and you'll bump the priority.
